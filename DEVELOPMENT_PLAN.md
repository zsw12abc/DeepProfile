# DeepProfile Chrome Extension 开发文档

本文档旨在指导 AI 助手一步步完成 DeepProfile Chrome 插件的开发。本项目用于在知乎页面分析用户画像，辅助理性交流。

## 1. 项目概述
*   **项目名称**: DeepProfile
*   **核心功能**: 监听知乎评论区 -> 提取用户 ID -> 抓取用户动态 -> 调用 LLM 分析 -> 展示用户画像。
*   **目标平台**: Chrome 浏览器 (Manifest V3)。

## 2. 技术栈规范
*   **语言**: TypeScript (必须，确保类型安全)。
*   **构建工具**: Vite 或 Plasmo (推荐 Plasmo，因为它对 Chrome 插件开发支持最好)。
*   **UI 框架**: React 或 Preact (轻量级)。
*   **测试框架**: Vitest (单元测试)。
*   **包管理器**: pnpm 或 npm。

## 3. 敏感数据与安全
*   **原则**: 任何 API Key 均不得硬编码在代码中。
*   **存储**: 所有敏感配置（API Key, Base URL）必须存储在 `chrome.storage.local` 中。
*   **版本控制**: `.env` 文件及包含密钥的测试文件必须加入 `.gitignore`。

### 阶段五：基础功能完善 (🚧 当前重点)
**目标**: 完善核心体验，实现历史记录与导出功能。

1.  **历史记录与缓存 (History Service)**:
    *   **需求**: 分析过的用户画像应存储在本地 (`chrome.storage.local`)。
    *   **逻辑**: 
        *   再次点击已分析用户时，直接从本地读取并显示，不消耗 Token。
        *   提供“重新分析”按钮以强制刷新。
        *   设置缓存有效期 (如 24 小时)。
    *   **UI**: 在设置页或单独页面提供“历史记录管理”，允许查看或删除记录。
2.  **导出与分享 (Export Service)**:
    *   **需求**: 用户可以将分析结果保存下来。
    *   **功能**:
        *   导出为 Markdown 文本 (方便复制)。
        *   导出为图片 (生成精美的分享卡片)。
3.  **UI/UX 细节优化**:
    *   优化 Loading 动画。
    *   完善错误提示 (网络错误、Key 无效等)。

## 5. 给 AI 的开发指令示例

当你开始开发时，请遵循以下模式：

> "我现在开始开发**阶段五**。首先，我将创建 `HistoryService.ts` 用于管理本地缓存。接着，我将更新 `ProfileService` 以在分析前检查缓存。"
